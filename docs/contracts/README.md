# QNetBench v0.1 Contracts

These documents and the JSON Schemas under `schemas/v0_1/` are normative for the 0.1 alpha line. Examples are informative unless a test names them as fixtures.

- `benchmark_spec_v0_1.md`: backend-independent scientific input.
- `canonical_result_v0_1.md`: run manifest, terminal request records, bundles, and failure behavior.
- `metric_definitions_v0_1.md`: stable metric IDs, row columns, populations, and unavailable states.

Breaking changes require explicit owner approval, an ADR, updated fixtures, and a versioning decision.
